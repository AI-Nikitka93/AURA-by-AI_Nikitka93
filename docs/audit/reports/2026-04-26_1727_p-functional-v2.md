## Claimed But Not Proven
- Публичная форма early access выглядит как рабочий lead-capture flow, но наружу заявки сейчас не уходят: live worker отвечает `mode: device`, а POST возвращает `waitlist_not_configured`.
- Companion/app narrative присутствует в copy и advisor output, но реального companion app, device sync, notifications или account layer нет.
- Offline/PWA surface задекларирован manifest-ом и install copy, но надёжная install/offline capability не доказана и фактически заглушена кодом.
- Контакт `privacy@aura-by-ai.com` указан в `privacy.html`, но отдельно не подтверждён как рабочий operator channel.

## Missing Expected Capabilities

  Реальная доставка early-access заявок в operator path отсутствует. Для публичной формы это главный functional gap.

## Highest-Impact Improvements

### Critical Gaps
1. Нет gaps уровня `Critical for stated project identity`: как portfolio case проект уже показывает реальный frontend surface, privacy controls и live AI-слой.

### Critical Only For Full Productization
1. Подключить живой waitlist relay (`Resend` или webhook/CRM), чтобы форма перестала быть device-only queue. Зачем: это единственный способ превратить текущую CTA surface в реальный lead flow. Сложность: 💛 Средне
2. Либо полноценно вернуть install/offline flow, либо убрать PWA/install promise из UX и metadata. Зачем: сейчас capability выглядит как наличие, но фактически выключена. Сложность: 💛 Средне

### Before Release
1. Добавить явный manual retry/sync control для локальной очереди в `PrivacyControlCenter`, а не полагаться только на автоматический flush. Зачем: повышает полезность queue fallback и делает статус прозрачным. Сложность: 💚 Легко
2. Показать `lastSubmission` и последний sync/result статус в privacy center. Зачем: пользователь увидит, была ли заявка реально отправлена или осталась локальной. Сложность: 💚 Легко
3. Добавить import/recover flow для экспортированного JSON либо честно маркировать export как one-way support artifact. Зачем: сейчас export есть, но восстановление состояния не предусмотрено. Сложность: 💛 Средне

### High ROI Additions
1. Связать generated AI brief с waitlist payload или downloadable artifact. Зачем: это самый сильный differentiated surface проекта, и его можно превратить из ephemeral текста в сохраняемый user asset. Сложность: 💚 Легко
2. Развести в UI `concept narrative` и `implemented capability`: например, явно помечать device-only queue и conceptual companion language. Зачем: уменьшает mismatch между copy и реальными возможностями. Сложность: 💚 Легко

### Optional / Later
1. Реализовать лёгкий import из share/export data для восстановления ritual profile на новом устройстве. Зачем: усилит privacy-first/local-first story. Сложность: 💛 Средне
2. Добавить operator-facing relay health endpoint/status badge с последним успешным relay mode. Зачем: снизит ручную диагностику для maintainer. Сложность: 💛 Средне

### Do Not Build
- Не добавлять auth/account layer, admin panel или checkout, пока проект остаётся честным portfolio case без реального commerce promise.
- Не строить полноценный native companion app раньше, чем заработает базовый lead capture и появится реальный продуктовый validation loop.
